۱. ساختار پروژه

amoozeshyar/
├── config/           # تنظیمات Django (settings, urls)
├── core/             # مدل‌ها، migrations، admin
│   ├── models.py     # تمام جداول دیتابیس
│   ├── admin.py      # ثبت پنل ادمین (فعلاً ConstValue)
│   └── migrations/

├── db.sqlite3        # بعد از migrate ساخته می‌شود
└── venv/             # محیط مجازی پایتون

۲. اجرای پروژه (گام‌به‌گام)

پیش‌نیاز

  • Python 3.11 یا بالاتر
  • محیط مجازی venv (از قبل در پروژه وجود دارد)

فعال‌سازی محیط مجازی

ویندوز (PowerShell):

cd c:\amoozeshyar
.\venv\Scripts\Activate.ps1

ویندوز (CMD):

cd c:\amoozeshyar
venv\Scripts\activate.bat

لینوکس / macOS:

cd /path/to/amoozeshyar
source venv/bin/activate

نصب وابستگی‌ها (در صورت نیاز)

pip install django

ساخت دیتابیس

مدل‌ها در اپ core تعریف شده‌اند. برای ایجاد جداول:

python manage.py makemigrations core
python manage.py migrate

فایل db.sqlite3 در ریشه پروژه ایجاد می‌شود.

ساخت کاربر ادمین

python manage.py createsuperuser

نام کاربری، ایمیل و رمز عبور را وارد کنید.

اجرای سرور توسعه

python manage.py runserver
آدرستوضیح
http://127.0.0.1:8000/admin/پنل مدیریت Django
http://127.0.0.1:8000/docs/همین راهنما (در مرورگر)
تغییر مدل‌ها: هر بار که core/models.py را ویرایش کردید، دوباره makemigrations و migrate را اجرا کنید.

۳. پنل ادمین ConstValue (مقادیر ثابت)

جدول const_values در مدل ConstValue تعریف شده و در core/admin.py با امکانات زیر ثبت شده است:

قابلیتشرح
جستجوعنوان، کد، نام، مقدار، عنوان مقدار ثابت، والد و فرزندان؛ جستجوی عددی با شناسه (id)
فیلتر وضعیتفعال / غیرفعال
فیلتر سلسله‌مراتبریشه، زیرمجموعه، دارای فرزند، برگ
فیلتر والدلیست والدهایی که فرزند دارند
فیلتر فیلدهای خالینام، مقدار، عنوان و …
نمایش لیستسلسله‌مراتب بصری، لینک والد، تعداد فرزندان
عملیات گروهیفعال‌سازی / غیرفعال‌سازی دسته‌ای
فرمگروه‌بندی فیلدها + autocomplete برای انتخاب والد

مسیر در ادمین: Core → مقادیر ثابت

۴. ساخت پنل ادمین برای جداول دیگر

برای هر مدل در core/models.py می‌توانید همان الگوی ConstValue را تکرار کنید.

گام ۱ — import مدل

در core/admin.py:

from .models import ConstValue, Person, Student  # مدل‌های مورد نیاز

گام ۲ — ثبت ساده (حداقل)

@admin.register(Person)
class PersonAdmin(admin.ModelAdmin):
    list_display = ("id", "username", "national_code", "full_name", "is_active")
    list_filter = ("is_active", "gender", "is_foreign")
    search_fields = ("username", "national_code", "first_name", "last_name", "full_name")
    list_per_page = 50

گام ۳ — فیلتر سفارشی (اختیاری)

برای شرط‌های خاص (مثل ConstValue) از admin.SimpleListFilter استفاده کنید:

class ActiveStudentsFilter(admin.SimpleListFilter):
    title = "وضعیت"
    parameter_name = "active"

    def lookups(self, request, model_admin):
        return [("yes", "فعال"), ("no", "غیرفعال")]

    def queryset(self, request, queryset):
        if self.value() == "yes":
            return queryset.filter(is_active=True)
        if self.value() == "no":
            return queryset.filter(is_active=False)
        return queryset

گام ۴ — روابط ForeignKey

  • autocomplete_fields = ("parent", "person") — انتخاب سریع با جستجو
  • raw_id_fields = ("person",) — پنجره انتخاب با id
  • list_select_related = ("person",) — کاهش کوئری در لیست
  • list_filter = ("person", admin.RelatedOnlyFieldListFilter) — فقط مقادیر مرتبط

گام ۵ — اینلاین برای جداول وابسته

مثلاً شماره تلفن زیر Person:

class PhoneNumberInline(admin.TabularInline):
    model = PhoneNumber
    extra = 1

@admin.register(Person)
class PersonAdmin(admin.ModelAdmin):
    inlines = [PhoneNumberInline]

گام ۶ — ستون محاسباتی در لیست

@admin.display(description="نام نمایشی")
def display_name(self, obj):
    return obj.full_name or obj.username

چک‌لیست برای هر مدل جدید

  1. مدل در core/models.py با verbose_name فارسی
  2. کلاس ModelAdmin در core/admin.py
  3. list_display — ستون‌های مهم
  4. search_fields — فیلدهای متنی قابل جستجو
  5. list_filter — بولین، choice، تاریخ، FK
  6. در صورت FK زیاد: autocomplete_fields + search_fields روی مدل والد
  7. اجرای migrate در صورت تغییر مدل
  8. تست در /admin/

نمونه ثبت چند مدل

from django.contrib import admin
from .models import ConstValue, Person, Student

# ConstValue قبلاً ثبت شده

@admin.register(Person)
class PersonAdmin(admin.ModelAdmin):
    search_fields = ("username", "national_code", "first_name", "last_name")
    list_filter = ("is_active", "gender")
    list_display = ("username", "national_code", "full_name", "is_active")

@admin.register(Student)
class StudentAdmin(admin.ModelAdmin):
    autocomplete_fields = ("person", "tendency", "advisor_teacher")
    search_fields = ("person__username", "student_number")
    list_display = ("id", "person", "student_number", "is_active")
برای autocomplete_fields روی مدل والد (مثلاً Person) حتماً search_fields در PersonAdmin تعریف کنید؛ در غیر این صورت autocomplete کار نمی‌کند.
